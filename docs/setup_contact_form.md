# 質問・要望フォームのセットアップ手順書

このドキュメントは、管理者（サイトオーナー）が質問・要望フォーム（contact ページ）のバックエンドとして動作する Google Apps Script (GAS) ウェブアプリを構築・デプロイするための手順書です。

## 1. Google スプレッドシートの作成
1. Google ドライブでスプレッドシートを新規作成します。
2. スプレッドシート名を入力します（例: `ながのデータ探検隊 質問・要望`）。
3. 最初のシート（左端のシート）の 1 行目に以下のヘッダー（列名）を設定します。
   - A列: `受信日時`
   - B列: `種別`
   - C列: `名前`
   - D列: `メール`
   - E列: `内容`
   - F列: `ページ`
4. ブラウザのURLからスプレッドシートID（`/d/` と `/edit` の間にある長い文字列）をコピーし、控えておきます。
   - 例: `https://docs.google.com/spreadsheets/d/【スプレッドシートID】/edit...`

## 2. Google Apps Script の設定
1. スプレッドシートのメニューバーから「拡張機能」→「Apps Script」を選択します。
2. デフォルトで作成されている `コード.gs` 内のコードをすべて削除し、リポジトリの [gas_contact_form.gs](gas_contact_form.gs) の内容をすべて貼り付けます。
3. コード冒頭にある以下の設定変数を書き換えます。
   - `const SHEET_ID = '控えたスプレッドシートID';`
   - `const NOTIFY_EMAIL = '通知を受け取るメールアドレス';`
4. ツールバーの保存アイコンをクリックするか、`Ctrl + S`（Macは `Cmd + S`）でプロジェクトを保存します。

## 3. ウェブアプリのデプロイ
1. 画面右上の「デプロイ」ボタンをクリックし、「新しいデプロイ」を選択します。
2. 種類の選択（歯車アイコン）で「ウェブアプリ」を選択します。
3. 以下の設定項目を入力・選択します。
   - 説明: 任意の識別名（例: `v1`）
   - 次のユーザーとして実行: **自分** (Your Google account)
   - アクセスできるユーザー: **全員** (Anyone)
     > [!IMPORTANT]
     > 「アクセスできるユーザー」を「全員」にしないと、サイトのフォームから送信したリクエストが権限エラーになります。
4. 「デプロイ」をクリックします。
5. 初回デプロイ時は「アクセスの承認」を求められるため、「アクセスを承認」をクリックし、自身のGoogleアカウントを選択します。安全警告が表示された場合は「詳細」→「ながのデータ探検隊（安全ではないページ）に移動」をクリックし、許可してください。
6. デプロイ完了画面に表示される「ウェブアプリ URL」（`https://script.google.com/macros/s/.../exec` で終わるURL）をコピーします。

## 4. フロントエンドの設定
1. リポジトリ内の `contact/index.html` を開きます。
2. JSコード内の `GAS_ENDPOINT_URL` の値を、コピーしたウェブアプリ URL に書き換えます。
   ```javascript
   const GAS_ENDPOINT_URL = 'https://script.google.com/macros/s/.../exec';
   ```
3. 変更を保存し、リポジトリをデプロイします。

---

## 注意：コード修正時の再デプロイについて
GAS 側のコードを変更した場合は、単に保存するだけでは既存のウェブアプリ URL に反映されません。以下の手順で更新する必要があります。

1. GAS エディタでコードを修正し、保存します。
2. 画面右上の「デプロイ」→「デプロイを管理」をクリックします。
3. アクティブなデプロイを選択し、鉛筆アイコン（編集）をクリックします。
4. バージョン項目で「**新バージョン**」を選択します。
5. 「デプロイ」をクリックします。これにより、同じウェブアプリ URL のまま変更が反映されます。
